# 首页上传日志入口设计

## 背景

`/` 首页当前在头部区域展示了“上传日志”主按钮，但该按钮没有任何跳转或行为。项目中已经存在完整的数据导入页面 `/imports`，因此首页入口与实际导入流程尚未连通。

## 目标

让首页“上传日志”按钮成为一个可用入口，点击后进入 `/imports`，帮助用户从首页快速进入数据导入流程。

## 非目标

- 不新增上传弹窗
- 不修改 `/imports` 页面逻辑
- 不新增客户端状态或交互逻辑

## 方案选型

### 方案 A：改为链接到 `/imports`

将首页头部的“上传日志”按钮替换为 `Link` 组件，直接跳转到 `/imports`。

优点：
- 与现有 Next.js 导航方式一致
- 改动最小
- 不引入额外客户端逻辑
- 与当前已有导入页结构完全匹配

### 方案 B：保留按钮并使用 `router.push("/imports")`

可以实现同样效果，但需要客户端事件处理，复杂度高于纯导航链接。

### 方案 C：点击后打开上传弹窗

不采用。仓库中已有独立导入页，再做一套首页弹窗会重复建设且容易偏离当前信息架构。

## 最终设计

采用方案 A。

在首页文件中将无行为的“上传日志”按钮替换为跳转到 `/imports` 的 `Link`，保留现有按钮样式和文案不变。这样首页首屏 CTA 与现有导入流程连通，同时不会影响其他页面逻辑。

## 验证方式

- 运行 `npm run lint`
- 人工检查首页按钮代码，确认目标路径为 `/imports`
